Qwen3Model::build_graph

从输入 token_ids 到输出 logits 的完整计算图构建过程

◈
总览: build_graph 全流程
Qwen3Model::build_graph(const GGUFInfo& info) → unique_ptr<ComputeGraph>
▶
input_ids [1, seq_len] | v [Step 1] parse_config() -- 从 GGUF 元数据提取模型超参 [Step 2] 创建 input_ids 占位符 -- 计算图入口 [Step 3] 创建 rope_cos/sin 缓存 -- 旋转位置编码预计算表 [Step 4] embedding_lookup -- token → hidden 向量 [Step 5] 28x Transformer Layer -- 核心计算 (Attention + FFN) [Step 6] final_norm + LM Head -- 归一化 + 词表投影 [Step 7] build_from_outputs({logits}) -- 反向遍历构建图 | v logits [1, seq_len, 151936]
整个函数只构建计算图（声明式），不执行任何计算。所有 Tensor 的 data 指针为 nullptr，实际内存由调度器在 schedule() 阶段分配。

Self-Attention 分支 + SwiGLU FFN 分支，双重残差连接
▶
▶ Self-Attention 分支
1.1 Pre-Attention RMSNorm
// blk.{i}.attn_norm.weight [1024] Tensor* x_norm = OpFactory::rms_norm(input, attn_norm_info, eps, "x_norm", i); // [1, S, 1024] → [1, S, 1024] // 公式: y = x / sqrt(mean(x^2) + eps) * weight
1.2 Q / K / V 线性投影
Tensor* q_flat = OpFactory::linear(x_norm, q_weight, "q_flat", i, true); // [1, S, 1024] @ [2048, 1024]^T → [1, S, 2048] (16 heads x 128) Tensor* k_flat = OpFactory::linear(x_norm, k_weight, "k_flat", i, true); // [1, S, 1024] @ [1024, 1024]^T → [1, S, 1024] (8 kv_heads x 128) Tensor* v_flat = OpFactory::linear(x_norm, v_weight, "v_flat", i, true); // [1, S, 1024] @ [1024, 1024]^T → [1, S, 1024] (8 kv_heads x 128)
transpose=true: 权重存储为 [out, in]，数学上等价于 x @ W^T。实际实现时不物理转置，而是调整访问模式。
1.3 Reshape + Permute → 4D
Tensor* q_4d = OpFactory::reshape_permute(q_flat, {1, -1, num_heads, head_dim}, // [1, S, 16, 128] {0, 2, 1, 3}, // → [1, 16, S, 128] "q_4d", i); Tensor* k_4d = OpFactory::reshape_permute(k_flat, {1, -1, num_kv_heads, head_dim}, // [1, S, 8, 128] {0, 2, 1, 3}, // → [1, 8, S, 128] "k_4d", i);
Permute {0,2,1,3} 将 seq_len 维度移到第 2 位，heads 移到第 1 位，形成 [B, heads, seq, head_dim] 布局，适配注意力计算。
1.4 Per-head RMSNorm
// Qwen3 特有: 对每个 head 的 head_dim 维度做 RMSNorm Tensor* q_normed = OpFactory::rms_norm(q_4d, q_norm_info, eps, "q_normed", i); // [1, 16, S, 128], weight=[128] → [1, 16, S, 128] Tensor* k_normed = OpFactory::rms_norm(k_4d, k_norm_info, eps, "k_normed", i); // [1, 8, S, 128], weight=[128] → [1, 8, S, 128]
1.5 旋转位置编码 (RoPE)
auto [q_rope, k_rope] = OpFactory::apply_rope( q_normed, k_normed, rope_cos, rope_sin, nullptr, "", i); // 对 head_dim 维度做旋转: [cos -sin; sin cos] * [q_even, q_odd] // 注入位置信息，使模型感知 token 顺序
1.6 Scaled Dot-Product Attention (GQA)
Tensor* attn_4d = OpFactory::Attention( q_rope, k_rope, v_4d, nullptr, // mask (因果mask在后端生成) 1.0f/sqrt(128.0f), // scale = 1/sqrt(head_dim) true, // causal = true 16/8, // num_kv_groups = 2 (GQA) "attn_4d", OP_TYPE_PAGED_ATTN, // 支持 PagedAttention i); // Q [1,16,S,128] @ K^T [1,8,S,128] → attn [1,16,S,S] @ V [1,8,S,128] // → [1, 16, S, 128]
GQA: 16 个 Q 头分成 8 组，每组 2 个 Q 头共享 1 个 KV 头。K/V 会被 repeat 2 次再计算注意力。
1.7 合并多头 → 输出投影 → 残差
// [1, 16, S, 128] → permute{0,2,1,3} → [1, S, 16, 128] → reshape → [1, S, 2048] Tensor* attn_flat = OpFactory::permute_reshape(attn_4d, {0,2,1,3}, {1, -1, num_heads * head_dim}, "attn_flat", i); // [1, S, 2048] @ [1024, 2048]^T → [1, S, 1024] Tensor* attn_out = OpFactory::linear(attn_flat, attn_out_weight, "attn_out", i, true); // 残差连接 1 Tensor* ffn_input = OpFactory::add(attn_out, input, "ffn_input", i); // [1, S, 1024] + [1, S, 1024] → [1, S, 1024]
▶ SwiGLU FFN 分支
2.1 Post-Attention RMSNorm
Tensor* ffn_normed = OpFactory::rms_norm(ffn_input, ffn_norm_info, eps, "ffn_normed", i); // blk.{i}.ffn_norm.weight [1024] // [1, S, 1024] → [1, S, 1024]
2.2 SwiGLU: Gate + Up 并行投影
// gate: [1, S, 1024] @ [3072, 1024]^T → [1, S, 3072] Tensor* gate = OpFactory::linear(ffn_normed, gate_weight, "gate", i, true); // up: [1, S, 1024] @ [3072, 1024]^T → [1, S, 3072] Tensor* up = OpFactory::linear(ffn_normed, up_weight, "up", i, true);
2.3 SiLU 激活 + 逐元素乘
// SwiGLU: SiLU(gate) * up Tensor* gate_act = OpFactory::silu(gate, "gate_act", i); // SiLU(x) = x * sigmoid(x) Tensor* ffn_inter = OpFactory::mul(gate_act, up, "ffn_inter", i); // [1, S, 3072] * [1, S, 3072] → [1, S, 3072] (逐元素)
2.4 Down 投影 + 残差连接 2
// [1, S, 3072] @ [1024, 3072]^T → [1, S, 1024] Tensor* ffn_out = OpFactory::linear(ffn_inter, down_weight, "ffn_out", i, true); // 残差连接 2: FFN 输出 + Attention 残差输出 Tensor* layer_output = OpFactory::add(ffn_out, ffn_input, "layer_output", i); // [1, S, 1024] + [1, S, 1024] → [1, S, 1024]
单层完整数据流
input [1, S, 1024] | v | RMSNorm [1, S, 1024] | 残差 1 | | +---> Linear(Q) --> [1,S,2048] --> reshape_permute --> [1,16,S,128] | | +---> Linear(K) --> [1,S,1024] --> reshape_permute --> [1, 8,S,128] | | +---> Linear(V) --> [1,S,1024] --> reshape_permute --> [1, 8,S,128] | | | Per-head RMSNorm (Q, K) | | | | | Apply RoPE (Q, K) | | | | | GQA Attention | | [1, 16, S, 128] | | | | | permute_reshape | | [1, S, 2048] | | | | | Linear(attn_out) | | [1, S, 1024] | | | | +---------> add <-----------+------------------------------+ | v ffn_input [1, S, 1024] | v | 残差 2 RMSNorm [1, S, 1024] | | | +---> Linear(gate) --> [1, S, 3072] --+--> SiLU --> mul --+ | | | +---> Linear(up) --> [1, S, 3072] --+-------------------+ | ffn_inter [1, S, 3072] | Linear(down) | ffn_out [1, S, 1024] | add <---+ | layer_output [1, S, 1024]

build_from_outputs - 反向构建计算图
graph->build_from_outputs({logits})
▶
// 从输出张量 logits 出发，沿 src[] 指针反向遍历整个 DAG auto graph = std::make_unique<ComputeGraph>(); graph->build_from_outputs({logits}); return graph;
build_from_outputs 从 logits 开始 BFS/DFS 遍历所有 src[] 依赖，收集所有参与计算的 Tensor 节点，构建完整的有向无环图 (DAG)。

OpFactory 算子概览
所有算子只声明形状和依赖，不执行计算
▶
算子函数输入 → 输出用途
placeholderOpFactory::placeholder无 → [B, S]输入占位
embedding_lookupOpFactory::embedding_lookup[B,S] + [V,H] → [B,S,H]词嵌入
linearOpFactory::linear[...,in] @ [out,in]^T → [...,out]矩阵乘
rms_normOpFactory::rms_norm[...] → [...]归一化
reshape_permuteOpFactory::reshape_permute[B,S,N,D] → [B,N,S,D]头拆分
permute_reshapeOpFactory::permute_reshape[B,N,S,D] → [B,S,N*D]头合并
apply_ropeOpFactory::apply_ropeQ,K + cos,sin → Q',K'位置编码
AttentionOpFactory::AttentionQ,K,V → [B,N,S,D]GQA 注意力
siluOpFactory::silu[...] → [...]SiLU 激活
mulOpFactory::mul[...] * [...] → [...]逐元素乘
addOpFactory::add[...] + [...] → [...]残差连接
所有算子创建的 Tensor 的 data = nullptr，op_type 记录操作类型。实际计算在 Executor::forward() 中由后端 kernel 执行。

阶段对 Tensor 的影响
build_graph创建所有节点，data=nullptr，仅定义形状和依赖关系
schedule为每个 Tensor 分配设备 (CPU/GPU)，计算内存需求，创建 MemoryPool
load_weights从 GGUF 文件读取权重数据，绑定到 WEIGHT 类型 Tensor 的 data 指针
generate绑定 input_ids 数据，为 ACT/CACHE 分配运行时内存，执行前向计算
