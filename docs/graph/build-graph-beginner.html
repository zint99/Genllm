🧩 图解 Qwen3 计算图构建

从一行代码开始，看看 AI 模型是如何被"翻译"成计算图的

Qwen3Model::build_graph() 函数的执行过程

计算图是什么？

想象你在做一道菜：番茄炒蛋

你需要：

  • 食材：番茄、鸡蛋（对应：输入数据、模型权重）
  • 步骤：打蛋→炒蛋→切番茄→翻炒（对应：各种运算操作）
  • 顺序：必须先打蛋才能炒蛋（对应：数据依赖关系）

计算图就是把这些步骤画成一张"流程图"，告诉计算机：

  • 需要什么数据
  • 按什么顺序计算
  • 每个计算的输入输出是什么

1 解析模型配置

▼

首先，我们需要知道这个模型有多大、有多少层、每层有多少神经元。

就像做菜前要先看菜谱，了解需要多少食材。

// 从 GGUF 文件中读取模型配置 this->parse_config(info); // 读取到的配置： config_.hidden_size = 1024; // 隐藏层维度 config_.num_layers = 28; // Transformer 层数 config_.num_heads = 16; // 注意力头数 config_.num_kv_heads = 8; // KV 头数 (GQA) config_.head_dim = 128; // 每个头的维度 config_.vocab_size = 151936; // 词表大小

📊 这些配置决定了什么？

  • hidden_size = 1024：每个词被表示为 1024 维的向量
  • num_layers = 28：模型有 28 层 Transformer，像 28 个加工车间
  • num_heads = 16：注意力机制有 16 个"视角"同时看问题
  • vocab_size = 151936：词表有 15 万多个词

2 创建输入节点

▼

计算图的起点是输入数据。我们先创建一个"占位符"，告诉图："这里将来会放输入"。

就像做菜前，先把盘子摆好，等菜做好了再装盘。

// 创建输入占位符 Tensor* input_ids = OpFactory::placeholder( DataType::GGML_TYPE_I32, // 数据类型：整数 TensorType::TENSOR_TYPE_INPUT, // 类型：输入 {1, -1}, // 形状：[1, seq_len] "input_ids", // 名字 -1 // layer_id );

🔍 计算图当前状态

📥 input_ids
形状: [1, seq_len]
等待输入的 token ID 序列
为什么要用 -1？
seq_len = -1 表示"动态长度"。就像一个可伸缩的盘子，不管放多少菜都能装得下。

3 词嵌入层 (Embedding)

▼

计算机不认识文字，只认识数字。Embedding 就是把数字 ID 变成有意义的向量。

就像把"番茄"这个词，变成一串描述它特征的数字：[红色, 圆形, 酸甜, 蔬菜...]

// 找到 Embedding 权重 const TensorInfo* embd_weight_info = OpFactory::find_tensor(info, "token_embd.weight"); // 权重形状: [151936, 1024] (词表大小 × 隐藏维度) // 创建 Embedding 计算节点 Tensor* x_in = OpFactory::embedding_lookup( input_ids, // 输入: [1, seq_len] embd_weight_info, // 权重: [151936, 1024] "x_in", -1, false ); // 输出: [1, seq_len, 1024]

🔍 计算图当前状态

📥 input_ids
[1, seq_len]
⬇️ 查表
🎯 Embedding
[151936, 1024] → [1, seq_len, 1024]

🔄 数据变化过程

  • 输入：[3, 15, 2048, 7] - 4 个词的 ID
  • 查表：从 15 万行的表中，取出第 3、15、2048、7 行
  • 输出：4 个 1024 维的向量，形状变为 [1, 4, 1024]

4 28 层 Transformer

▼

这是模型的核心部分！28 层 Transformer 就像 28 个加工车间，每个车间都做同样的事情，但用不同的"配方"（权重）。

// 循环创建 28 层 Transformer Tensor* prev_output = x_in; // [1, seq_len, 1024] for (int layer_idx = 0; layer_idx < 28; ++layer_idx) { prev_output = this->build_qwen3_layer( prev_output, // 上一层的输出 info, layer_idx, // 当前层编号 ... ); } // 每层: [1, seq_len, 1024] → [1, seq_len, 1024]

📥 input_ids

⬇️
🎯 Embedding
[1, seq_len, 1024]
⬇️
🔄 Transformer × 28
[1, seq_len, 1024] → [1, seq_len, 1024]
为什么形状不变？
Transformer 层的设计就是"输入输出同形状"，这样可以一层叠一层，像汉堡包一样。

4.1 单层 Transformer 详解

▼

每一层 Transformer 都包含两个主要部分：自注意力和前馈网络。

就像一个人：先"看"全局信息（注意力），再"想"怎么处理（前馈网络）。

🧠 自注意力 (Self-Attention) 详解

自注意力让每个词都能"看到"句子中的所有其他词，然后决定哪些词更重要。

// ====== Self-Attention 分支 ====== // 1. 先做归一化 (RMSNorm) Tensor* x_norm = OpFactory::rms_norm(input, ...); // [1, S, 1024] → [1, S, 1024] // 2. 计算 Q, K, V (三个不同的"视角") Tensor* q_flat = OpFactory::linear(x_norm, q_weight, ...); // Q (Query): "我在找什么？" [1, S, 2048] Tensor* k_flat = OpFactory::linear(x_norm, k_weight, ...); // K (Key): "我能提供什么？" [1, S, 1024] Tensor* v_flat = OpFactory::linear(x_norm, v_weight, ...); // V (Value): "我的具体内容是..." [1, S, 1024] // 3. 拆分成多个头 (16 个头) Tensor* q_4d = OpFactory::reshape_permute(q_flat, {1, -1, 16, 128}, // 拆成 16 个头 {0, 2, 1, 3}); // 调整维度顺序 // [1, S, 2048] → [1, 16, S, 128] // 4. 计算注意力分数 Tensor* attn_4d = OpFactory::Attention(q_rope, k_rope, v_4d, ...); // Q @ K^T → 注意力权重 → 加权求和 V // [1, 16, S, 128] // 5. 合并多头，输出投影 Tensor* attn_flat = OpFactory::permute_reshape(attn_4d, ...); // [1, 16, S, 128] → [1, S, 2048] Tensor* attn_out = OpFactory::linear(attn_flat, attn_out_weight, ...); // [1, S, 2048] → [1, S, 1024] // 6. 残差连接 Tensor* ffn_input = OpFactory::add(attn_out, input, ...); // [1, S, 1024] + [1, S, 1024] → [1, S, 1024]

🔮 前馈网络 (FFN) 详解

FFN 使用 SwiGLU 激活函数，是 Qwen3 的特色设计。

// ====== SwiGLU FFN 分支 ====== // 1. 归一化 Tensor* ffn_normed = OpFactory::rms_norm(ffn_input, ...); // 2. 两条并行路径 (Gate 和 Up) Tensor* gate = OpFactory::linear(ffn_normed, gate_weight, ...); // [1, S, 1024] → [1, S, 3072] Tensor* up = OpFactory::linear(ffn_normed, up_weight, ...); // [1, S, 1024] → [1, S, 3072] // 3. SwiGLU 激活: SiLU(gate) × up Tensor* gate_act = OpFactory::silu(gate, ...); // SiLU(x) = x × sigmoid(x)，一种平滑的激活函数 Tensor* ffn_inter = OpFactory::mul(gate_act, up, ...); // [1, S, 3072] × [1, S, 3072] → [1, S, 3072] // 4. 输出投影 + 残差 Tensor* ffn_out = OpFactory::linear(ffn_inter, down_weight, ...); // [1, S, 3072] → [1, S, 1024] Tensor* layer_output = OpFactory::add(ffn_out, ffn_input, ...); // [1, S, 1024] + [1, S, 1024] → [1, S, 1024]

🤔 为什么用 SwiGLU？

传统的 FFN 用 ReLU 激活函数，SwiGLU 是一种更先进的设计：

  • Gate (门控)：决定哪些信息重要
  • Up (上采样)：把信息放大到更高维度
  • SiLU 激活：比 ReLU 更平滑，效果更好
  • 乘法融合：门控 × 上采样，保留重要信息

5 输出层 (LM Head)

▼

经过 28 层 Transformer 后，我们需要把结果转换回"词的概率"。

就像做完菜后，要装盘上桌，让客人（用户）能看懂。

// 最终归一化 Tensor* final_norm = OpFactory::rms_norm( prev_output, // [1, seq_len, 1024] output_norm_info, ... ); // LM Head: 把 1024 维向量映射到 15 万个词 Tensor* logits = OpFactory::linear( final_norm, // [1, seq_len, 1024] embd_weight_info, // 复用 Embedding 权重！ "logits", ... ); // 输出: [1, seq_len, 151936]

🎯 Embedding
[1, seq_len, 1024]
⬇️
🔄 Transformer × 28
[1, seq_len, 1024]
⬇️
📏 RMSNorm
[1, seq_len, 1024]
⬇️
🎯 LM Head
[1, seq_len, 151936]
每个位置输出 15 万个词的概率
权重共享
LM Head 复用了 Embedding 的权重，这样可以减少参数量，提高效率。就像一本书的目录和索引用同一套词表。

6 构建计算图

▼

现在，我们从输出 logits 开始，反向遍历所有依赖，构建完整的计算图。

就像从"番茄炒蛋"这道菜开始，追溯需要哪些食材和步骤。

// 从 logits 开始，反向收集所有依赖节点 auto graph = std::make_unique<ComputeGraph>(); graph->build_from_outputs({logits}); return graph;

🔄 构建过程

  • 起点：logits 节点
  • 追溯：logits 依赖 final_norm，final_norm 依赖 Transformer 层...
  • 收集：所有参与计算的 Tensor 节点
  • 结果：一个完整的有向无环图 (DAG)

💡 关键理解

  • 声明式编程：build_graph 只"画图"，不"计算"
  • 延迟执行：所有 Tensor 的 data = nullptr，等调度器分配内存
  • 图优化：调度器会优化图结构，合并操作，分配设备
  • 高效推理：一次构建图，多次高效执行
