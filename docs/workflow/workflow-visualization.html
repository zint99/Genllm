🧠
智能调度算法
根据设备可用内存和算力，将模型层分配到最优设备，支持多GPU流水线
Step 1
估算每层内存开销: weight + activation + kv_cache
Step 2
贪心算法: 尽量将连续层分配到同一设备
Step 3
assign_global_nodes: Embedding/LMHead → CPU
Step 4
insert_copy_edges: 跨设备插入 D2H/H2D 拷贝
Step 5
create_memory_pools: 为每个设备创建内存池
Step 6
initialize_kv_cache: 预分配 KV Cache

9
自回归文本生成
RUNNING_TIME(executor.generate(tokens, 512, eos_id, &tokenizer))
🚀
生成流程
Prefill (并行处理prompt) → Decode (逐token生成)
Phase 1: Prefill
并行处理所有 prompt tokens → 生成第一个新 token
Phase 2: Decode Loop
自回归循环: forward() → sample() → decode_step()
采样策略
Top-p (核采样) + Temperature 控制随机性
终止条件
达到 max_tokens 或遇到 EOS token
🔄 Decode Loop 详解
1. resolve_dims(seq_pos, 1)
2. bind_input(token_id)
3. forward()
4. sample_top_p()
5. append_token()
6. seq_pos++
7. check EOS
// 生成主循环 std::vector<int32_t> result = executor.generate( tokens, // prompt token IDs 512, // max_new_tokens eos_id, // 终止 token &tokenizer // 用于实时打印生成文本 ); // 输出: 北京是中国的首都...
